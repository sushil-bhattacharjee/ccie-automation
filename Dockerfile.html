                  ↑                ↑                ↑

DNS Resolution

Docker's embedded DNS only resolves service names within the same network. Nginx cannot resolve the hostname mongodb because they don't share a network. Flask can resolve both nginx and mongodb because it sits on both networks.

Security benefit: Even if Nginx is compromised, the attacker cannot directly reach MongoDB.

CH.5

Volumes & Data Persistence

+

The Problem

Containers are ephemeral. When a container is destroyed, its writable layer (and all data) is lost. For MongoDB, this means all database records disappear on docker-compose down.

The Solution — Named Volumes

services:
  mongodb:
    image: mongo:6.0
    volumes:
      - mongo-data:/data/db         # Database files
      - mongo-config:/data/configdb # MongoDB configuration

volumes:
  mongo-data:
  mongo-config:

Volume Lifecycle

ConnectionConfigured InNetworkHow
Host → Nginxdocker-compose.ymlHost → frontendports: "8080:80"
Nginx → Flasknginx.conffrontendproxy_pass http://flask-app:5000
Flask → MongoDBapp.pybackendMongoClient("mongodb://mongodb:27017/")
Nginx → MongoDB❌ BLOCKEDNot on same networkCannot resolve hostname

Service Name = Hostname

Docker Compose automatically creates DNS entries for each service. The service name in docker-compose.yml becomes the hostname on the Docker network:

# docker-compose.yml service name → DNS hostname
flask-app  → resolves to flask-app's container IP
mongodb    → resolves to mongodb's container IP

This is why proxy_pass http://flask-app:5000 works — Docker DNS resolves flask-app to the correct container IP on the shared frontend network.

CH.7

Complete docker-compose.yml Reference

+

Production Configuration

version: '3.8'

services:

  nginx:
    build: ./nginx
    ports:
      - "${NGINX_HOST_PORT:-8080}:80"   # Only entry point
    networks:
      - frontend
    depends_on:
      - flask-app

  flask-app:
    build: ./flask-app
    env_file:
      - ./db/some_env_file.env          # Relative path, never absolute
    environment:
      MONGO_HOST: mongodb
      MONGO_PORT: 27017
      MONGO_DB: ${MONGO_DB_NAME}
    expose:
      - "5000"                          # Documentation only
    networks:
      - frontend
      - backend
    depends_on:
      - mongodb

  mongodb:
    image: mongo:6.0
    environment:
      MONGO_INITDB_ROOT_USERNAME: ${MONGO_INITDB_ROOT_USERNAME}
      MONGO_INITDB_ROOT_PASSWORD: ${MONGO_INITDB_ROOT_PASSWORD}
      MONGO_INITDB_DATABASE: ${MONGO_DB_NAME}
    expose:
      - "27017"                         # Documentation only
    networks:
      - backend
    volumes:
      - mongo-data:/data/db
      - mongo-config:/data/configdb

networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge

volumes:
  mongo-data:
  mongo-config:
