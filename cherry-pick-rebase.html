git cherry-pickSelective

Copy ONE specific commit to your current branch. Original stays in source branch.

# Copy single commit
git checkout release
git cherry-pick ba64768

# Copy multiple specific commits
git cherry-pick abc123 def456 ghi789

# Copy range (A through C inclusive)
git cherry-pick A^..C

# Apply without auto-commit
git cherry-pick --no-commit ba64768

Cherry-Pick Commands

CommandWhat It Does
git cherry-pick <hash>Copy one commit
git cherry-pick A B CCopy multiple specific commits
git cherry-pick A^..CCopy range A through C
git cherry-pick --no-commitApply changes, don't commit
git cherry-pick --continueAfter resolving conflict
git cherry-pick --abortCancel and revert
git cherry-pick --skipSkip conflicting commit
💡 Key: Cherry-pick COPIES (new hash). Original commit unchanged. Safe for shared branches.

git rebaseSync

Replay your commits on top of another branch. Creates linear history but rewrites commits.

# Sync feature with latest main
git checkout feature
git rebase main

# Interactive rebase - edit/squash/reorder
git rebase -i HEAD~4

# Continue after conflict resolution
git rebase --continue

# Abort and restore original state
git rebase --abort

Interactive Rebase Keywords

KeywordWhat It Does
pickKeep commit as-is
rewordKeep changes, edit message
editPause to amend content + message
squashMerge into above, combine messages
fixupMerge into above, discard this message
dropDelete commit entirely
⚠️ Warning: Rebase rewrites history! Never rebase commits that are pushed to shared branches.

git format-patch + git amOffline Transfer

Export commits as patch files for cross-repo or air-gapped transfer.

# Export last commit as patch
git format-patch -1 ba64768
# Creates: 0001-B2-Fix-SNMP.patch

# Export last 3 commits
git format-patch -3

# Apply patch AND create commit (preserves author)
git am 0001-B2-Fix-SNMP.patch

# Apply patch only (no commit)
git apply 0001-B2-Fix-SNMP.patch

🔧 Fixing Commit Mistakes

SituationCommandNotes
Fix last commit messagegit commit --amend -m "new msg"Rewrites last commit
Fix last commit contentgit add . && git commit --amendAdd forgotten files
Fix older commit messagegit rebase -i HEAD~N → rewordInteractive rebase
Fix older commit contentgit rebase -i HEAD~N → editPauses for amend
Undo last commit, keep stagedgit reset --soft HEAD~1Changes remain staged
Undo on shared branchgit revert HEADSafe - new commit
💡 Golden Rule: If pushed → use revert. If local → use amend, rebase -i, or reset.

📋 Release Workflow Example

# Scenario: Release needs ONE bugfix (B2) from bugfix branch

# 1. Cherry-pick specific fix to release
git checkout release
git cherry-pick ba64768   # Only B2, not B1 or B3

# 2. Sync release with main (get new features)
git rebase main             # Replay release commits on main

# 3. Merge release into main (final delivery)
git checkout main
git merge release           # Fast-forward if linear
