⚠️ The Critical Exam Trap

🎯 -s ours ≠ -X ours — They look similar but behave completely differently!
Aspect-s ours (Strategy)-X ours (Option)
Non-conflicting changes❌ Discarded✓ Merged normally
Conflicting changes❌ DiscardedAuto-pick our side
Their work preserved?No — thrown awayYes — only conflicts lost
Use caseAbandon branch cleanlyAuto-resolve in your favor
 
# STRATEGY: Discards ALL their changes (nuclear option)
git merge -s ours feature    # Nothing from feature is kept!

# OPTION: Only auto-resolves conflicts in our favor
git merge -X ours feature    # Non-conflicting changes still merge

👆 Ours vs Theirs

git checkout main           ← You're standing HERE = OURS
git merge feature           ← Pulling THIS in = THEIRS
💡 Remember: Ours = where I stand. Theirs = what I'm merging in.

📐 Understanding 3-Way Merge

"3-way" does NOT mean 3 branches. It means Git compares 3 snapshots of the file:

        M1  ← ANCESTOR (version 1) — where branches split
       /  \
    main    feature
  (version 2)  (version 3)
VersionWhat It IsWhere Git Finds It
1 — AncestorThe fork pointgit merge-base main feature
2 — OursBranch you're onHEAD
3 — TheirsBranch merging inThe branch name

Why Git Needs the Ancestor

# WITH ancestor (3-way — correct):
Ancestor:  snmp_poll("R1")       ← original
Ours:      snmp_poll("R1")       ← we DIDN'T change it
Theirs:    snmp_poll("SW1")      ← they DID change it

Git: "Only theirs changed → take theirs. No conflict."

# WITHOUT ancestor (2-way — wrong):
Ours:      snmp_poll("R1")
Theirs:    snmp_poll("SW1")

Git: "They're different... CONFLICT!"

🐙 Octopus Strategy

Merges more than 2 branches at once. Git selects it automatically when you pass multiple branches.

# Instead of 3 separate merges:
git merge feature-a
git merge feature-b
git merge feature-c

# Octopus — ONE merge commit, 3 parents:
git merge feature-a feature-b feature-c
⚠️ The Catch: Octopus refuses to handle conflicts. If any branch conflicts, it aborts entirely.
