⚙️ The Four NETCONF Operations

NETCONF operations control how configuration is applied. Specified via nc:operation attribute. Default = merge.

1. CREATE (like REST POST)

  • Behavior: Creates new configuration only
  • Fails if: Element already exists
  • Idempotent: ❌ No (fails on second run)
<access-list-seq-rule nc:operation="create">
    <sequence>45</sequence>
    <ace-rule>
        <action>permit</action>
        <protocol>tcp</protocol>
        <any/>
        <dst-any/>
    </ace-rule>
</access-list-seq-rule>

2. MERGE (like REST PATCH — Default)

  • Behavior: Creates new OR updates existing
  • Fails if: Never (always succeeds)
  • Idempotent: ✅ Yes (safe to run multiple times)
<access-list-seq-rule nc:operation="merge">
    <sequence>45</sequence>
    <ace-rule>
        <action>permit</action>
        <protocol>tcp</protocol>
        <any/>
        <dst-any/>
    </ace-rule>
</access-list-seq-rule>

3. REPLACE (like REST PUT)

  • Behavior: Replaces entire container/entry
  • Deletes: Everything not mentioned at that level
  • Idempotent: ✅ Yes
<extended nc:operation="replace">
    <name>MY-ACL</name>
    <access-list-seq-rule>
        <sequence>10</sequence>
        <ace-rule>
            <action>permit</action>
            <protocol>ip</protocol>
            <any/>
            <dst-any/>
        </ace-rule>
    </access-list-seq-rule>
</extended>
⚠️ Replace deletes unmentioned config! MY-ACL will have ONLY sequence 10. All other sequences are removed.

4. DELETE

  • Behavior: Removes specified elements
  • Fails if: Element doesn't exist
  • Idempotent: ❌ No (fails if already deleted)
<access-list-seq-rule nc:operation="delete">
    <sequence>45</sequence>
</access-list-seq-rule>

🔑 Key vs Data Elements

Keys (Identifiers): <name> = which ACL, <sequence> = which rule. Immutable — cannot change directly. No operations allowed on keys.
Data Elements: <action>, <protocol>, <host> — actual config values. Mutable. Inherit operations from parent.

To "change" a key (e.g., sequence 10→20), you must delete old + create new:

<access-list-seq-rule nc:operation="delete">
    <sequence>10</sequence>
</access-list-seq-rule>

<access-list-seq-rule nc:operation="create">
    <sequence>20</sequence>
    <ace-rule>
        <action>permit</action>
        <protocol>tcp</protocol>
        <any/>
        <dst-any/>
    </ace-rule>
</access-list-seq-rule>

📝 Jinja2 Template — ACL Payload Generator

The template config_acl.j2 renders NETCONF XML payloads from YAML data (acl_soth.yml).

Template (config_acl.j2)

<config xmlns="urn:ietf:params:xml:ns:netconf:base:1.0">
  <native xmlns="http://cisco.com/ns/yang/Cisco-IOS-XE-native">
    <ip>
      <access-list>
    {% for acl in ACCESS_LISTS.EXTENDED %}
        <extended xmlns="http://cisco.com/ns/yang/Cisco-IOS-XE-acl"
            xmlns:nc="urn:ietf:params:xml:ns:netconf:base:1.0"
            nc:operation="replace">
          <name>{{acl.NAME}}</name>
      {% for seq in acl.ENTRIES %}
          <access-list-seq-rule>
            <sequence>{{seq.SEQUENCE}}</sequence>
            <ace-rule>
              <action>{{seq.ACTION}}</action>
              <protocol>{{seq.PROTOCOL}}</protocol>
            {% if seq.SOURCE_ANY is defined and seq.SOURCE_ANY %}
              <any/>
            {% endif %}
            {% if seq.SRC_PORT_OP is defined %}
              <src-{{seq.SRC_PORT_OP}}>{{seq.SRC_PORT}}</src-{{seq.SRC_PORT_OP}}>
            {% endif %}
            {% if seq.SOURCE_HOST is defined %}
              <host-address>{{seq.SOURCE_HOST}}</host-address>
            {% endif %}
            {% if seq.DEST_NETWORK is defined %}
              <dest-ipv4-address>{{seq.DEST_NETWORK}}</dest-ipv4-address>
              <dest-mask>{{seq.DEST_WILDCARD}}</dest-mask>
            {% endif %}
            {% if seq.DEST_HOST is defined %}
              <dst-host-address>{{seq.DEST_HOST}}</dst-host-address>
            {% endif %}
            {% if seq.DEST_ANY is defined and seq.DEST_ANY %}
              <dst-any/>
            {% endif %}
            {% if seq.DEST_PORT_OP is defined %}
              <dst-{{seq.DEST_PORT_OP}}>{{seq.DEST_PORT}}</dst-{{seq.DEST_PORT_OP}}>
            {% endif %}
            </ace-rule>
          </access-list-seq-rule>
      {% endfor %}
        </extended>
    {% endfor %}
      </access-list>
    </ip>
  </native>
</config>

🏆 Key Takeaways

1. Operations target containers/lists, not keys.
2. Keys (<name>, <sequence>) are identifiers — immutable, no operations.
3. Data elements inherit operations from parent.
4. Default operation is merge if omitted.
5. To change a key → delete old + create new.
6. replace at ACL level deletes ALL unmentioned rules.
7. Never put replace on <access-list> container (affects ALL ACLs).
