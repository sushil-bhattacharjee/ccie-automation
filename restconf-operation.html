💡 JSON Payload Prefixes:
Container path: Cisco-IOS-XE-native:native/ip/access-list
ACL lists: Cisco-IOS-XE-acl:standard, Cisco-IOS-XE-acl:extended

⚡ ACL CRUD Brief

GET — can operate URI with list without key / list with key / container
POST — only container in URI and list with new key in the payload
PUT — URI and payload must be same, 201 for new resource and 204 if replaced
PATCH — URI and payload must be same, 204 for new or replaced
DELETE — can be container or list must be with key

     +--rw ip

           +--rw ios-acl:extended* [name]
              +--rw ios-acl:name                    ext-acl-type       ← KEY
              +--rw ios-acl:access-list-seq-rule* [sequence]
                 +--rw ios-acl:sequence    acl-sequence               ← KEY
                 +--rw ios-acl:ace-rule
                    +--rw ios-acl:action?                    acl-action
                    +--rw ios-acl:protocol?                  protocol-type
                    +--rw (ios-acl:source)?
                    |  +--:(any-source)       → <any/>
                    |  +--:(host-source)      → <host-address>
                    |  +--:(network-source)   → <ipv4-address> + <mask>
                    +--rw (ios-acl:src-port-match)?
                    |  +--:(src-equal)   → <src-eq>
                    |  +--:(src-gt)      → <src-gt>
                    |  +--:(src-lt)      → <src-lt>
                    +--rw (ios-acl:destination)?
                    |  +--:(any-destination)      → <dst-any/>
                    |  +--:(host-destination)     → <dst-host-address>

curl -i -k -X POST \

  --data '{
    "Cisco-IOS-XE-native:access-list": {

            "name": "Pod10_Task043",

        }]
    }
}'
❌ RESULT: FAILED — Payload contains parent container name

POST payload should contain only the child element (e.g., Cisco-IOS-XE-acl:standard), not the parent container name (Cisco-IOS-XE-native:access-list).

RFC 8040: The message-body MUST contain exactly one instance of the child resource to create within the parent.

  delete_acl()
elif operation=='post':
  post_acl()
elif operation=='put':
  put_acl()
elif operation=='patch':
  patch_acl()
else:
  print("❌ Invalid operation")

🏆 Key Takeaways

1. POST only works on empty containers — use PATCH for adding to existing data
2. PUT requires exact URI-to-payload match (container→container, list+key→list)
3. PATCH is most flexible — works with or without key in URI
4. DELETE on list requires key; on container deletes everything
5. Always use -i flag to see HTTP status codes (204 = success)
6. PATCH MUST NOT create target resource — target must exist
7. PATCH cannot delete child resources — use YANG Patch for delete
8. PUT replaces entirely — unmentioned sequences are deleted
