🐍 Python Reference — restconf_acl.py

A complete script driving CRUD against an IOS-XE RESTCONF endpoint for the extended ACL challenge-207-pod-10. Below is the same file split section-by-section.

1. Imports & SSL warning suppression

Standard library plus requests and urllib3; disabling InsecureRequestWarning lets us hit the device's self-signed cert without noisy stderr.

import os
import requests
import json
import urllib3
urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)

2. Credentials & headers

Pulled from environment variables. Both RESTCONF accept and content-type headers use the RFC 8040 hyphen form application/yang-data+json.

USERNAME=os.getenv("USERNAME")
PASSWORD=os.getenv("PASSWORD")
HOST=os.getenv("HOST_ADDRESS")
headers = {
'Accept': 'application/yang-data+json',
'Content-Type': 'application/yang-data+json',
}

3. Reusable ACL payload

The same dict is reused for POST, PUT, and PATCH. Note any: [None] and dst-any: [None] — Python None serialises to JSON null, which is how YANG empty-leaves are encoded under RFC 7951.

device_acl_payload={
  "Cisco-IOS-XE-acl:extended": {
    "name": "challenge-207-pod-10",
    "access-list-seq-rule": [
      {
        "sequence": "10",
        "ace-rule": {
          "action": "permit",
          "protocol": "tcp",
          "any": [None],
          "src-gt": 100,
          "dest-ipv4-address": "203.36.36.32",
          "dest-mask": "0.0.0.15"
        }
      },
      {
        "sequence": "20",
        "ace-rule": {
          "action": "deny",
          "protocol": "ip",
          "host-address": "198.51.100.6",
          "dst-any": [None]
        }
      }
    ]
  }
}

4. GET — read one extended ACL

URI ends with the list key =challenge-207-pod-10. No payload, just headers + basic auth.

def get_acl():
  #! TODO-1: Complete the url for GET to print extended acl "challenge-207-pod-10"
  url=f"https://{HOST}/restconf/data/Cisco-IOS-XE-native:native/ip/access-list/extended=challenge-207-pod-10"
  response=requests.get(url, headers=headers, auth=(USERNAME, PASSWORD), verify=False)
  print(f"\n{HOST}: RESPONSE STATUS CODE {response.status_code}")
  print(json.dumps(response.json(), indent=2))

5. DELETE — remove the ACL, then verify

DELETE never carries a body. The follow-up GET prints the status code only (no response.json()) because a successful delete returns 404 on the subsequent GET.

6. POST — create only (fails if ACL already exists)

URI is the parent container /access-list; the list key lives inside the payload. Fails with 409 Conflict if the ACL is already present.
