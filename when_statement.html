YANG Model

bgp-peering.yang 
module bgp-peering {
  namespace "http://example.com/bgp-peering";  prefix bp;
  import ietf-inet-types { prefix inet; }

  list bgp-peering {
    key "name";
    leaf name        { type string; }
    leaf local-as    { type uint32; }
    leaf remote-as   { type uint32; }
    leaf neighbor-ip { type inet:ipv4-address; }

    leaf peer-type {
      type enumeration {
        enum direct    { value 1; }
        enum untrusted { value 2; }
        enum transit   { value 3; }
    } }

    leaf as-type {
      type enumeration {
        enum customer { value 1; }
        enum provider { value 2; }
        enum peer     { value 3; }
    } }

    leaf-list allowed-ip-prefixes {
      type inet:ipv4-prefix;
      when "../peer-type = 'untrusted'";              // untrusted only
    }
    container static-routes {
      when "../peer-type = 'direct' or ../peer-type = 'untrusted'";
      leaf-list route { type inet:ipv4-prefix; }
    }
    container static-filter {
      when "../peer-type = 'untrusted' and ../as-type = 'customer'";
      leaf-list allowed-vlans { type uint16; }
      leaf strict-rpf { type boolean; default true; }
    }
    leaf max-prefix {
      type uint32;
      when "../peer-type != 'direct'";                // NOT direct
    }
    leaf import-policy {
      type string;
      when "../as-type = 'customer' or ../as-type = 'peer'";
    }
  }
}

Makes it worse — 2 violations → 3. import-policy now also breaks.

  • static-filter is the most restrictive — needs BOTH untrusted AND customer
  • provider kills both static-filter and import-policy
  • direct kills allowed-ip-prefixes, static-filter, and max-prefix
  • untrusted+customer is the most permissive — all five nodes allowed
  • transit kills allowed-ip-prefixes, static-routes, and static-filter

Exam Strategy

  1. Read peer-type and as-type first — before looking at any nodes
  2. Walk every when statement — evaluate TRUE/FALSE for that specific combo
  3. Check presence vs absence — node present + when=FALSE = violation. Node absent + when=FALSE = OK.
  4. Watch for unknown leaves — any key not in the YANG model = instant fail (Option B trap)
  5. For "fix" questions — verify the fix doesn't break something else (Option D makes it worse)
